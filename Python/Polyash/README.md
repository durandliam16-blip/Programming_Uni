Projet Polyash2025
============

Le projet, tirer d'un précedent concours de Google, 
consiste à trouver le parfait algorithme de gestion des feux verts/rouge.
En effet, cela est pour optimiser le temps total de trajet de toutes les voitures selon leur trajet prédefini. 
Tout les codes seront en python avec les bibliothèques de base (sauf si on veut visualiser les problème ou les solutions).
Notre objectif personnel sera de trouver une solution efficace et applicable dans la vrai vie. 

Le fichier `README.md` est écrit en [**Markdown**](https://docs.github.com/en/get-started/writing-on-github/getting-started-with-writing-and-formatting-on-github/basic-writing-and-formatting-syntax) et permet de soigner la _mise en forme_.


Fonctionnement du projet
========================

Tout d'abord, il faut récupérer les informations prédefinis (tel que les trajets, voitures et routes) en ouvrant le fichier texte fournit avec le code polyparser.
Ensuite, nous pourrons tester nos solutions avec le programme polysolver puis les soumettre à un score de performance sur le site officiel du concours. 
Enfin, une fois le projet fini nous devrons le mettre sous le meme format texte que le fichier d'entrée. Polyash sert à lancer l'ensemble.
Voir le google docs commun pour plus de détails. 
La syntaxe générale/minimale est
    polyash.py chemin/fichier_entree.txt chemin/fichier_sortie.txt

Voici nos 5 principales étapes pour nous guider : 
Etape 1 : Analyse du prblème, premières pistes et définition des roles  
Etape 2 : Mise en place du git 
Etape 3 : Test avec cas n'1
Etape 4 : Mise  à grande échelle pour les autres cas
Etape 5 : Rendu et vidéo de présentation de notre solution

Et les étapes de réflexion sont plus précisément : Analyse du problème + Décomposition en parties du problème + Définition de la structure data adapté + développement de l'algorithme + Traitement des entrées et sorties
